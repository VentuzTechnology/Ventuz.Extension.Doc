SceneBuilder API. Scene elements accessed by this API must be owned by a scene in Idle State. Usw the ModifyAsync(Action<VxScene>, params VxScene[]) method to schedule modifications not only to the right thread but also ensure the scene(s) to be in right state.
See also ../guide/vx_how_to_develop.html#threading

Export(VxNode[], Stream, VxExportFlags)

Exports one more more nodes into a VxSceneFragment. This export is the same procedure as storing items in the Repository of Designer All exported nodes must be on the same container level. It is recommended only to export multiple nodes of the same type (layer, hierarchy or content). Hierarchy nodes can include their children in the export. Children are are 'behind' a node, means connected to any output or any output of a child. It is also recommended that nodes passed as argument for 'nodes' are not nested, means no child of another node being exported.

VxSceneFragment Export(VxNode[] nodes, Stream stream, VxExportFlags flags)

Parameters

nodes VxNode[]

Nodes to export.

stream Stream

An optional writable stream to export the data. If no stream is specified, the export is performed in memory and the resulting data is available at the Data property.

flags VxExportFlags

Flags to define the kind of export

Returns

VxSceneFragment

A scene fragment containing information about the exported nodes. It also can contain the actual export data if the stream parameter was null.

ModifyAsync(VxSceneElement, Action<VxScene>, params VxScene[])

It is highly recommended to use this method to modify the structure of a scene. A single modification will result in an internal re-calculation of many internal structures so it is a better approach to combine multiple modifications together. Additionally the modification of a scene requires the scene in "Idle" state. If you like to modify a the during the validation of a scene or node, the scene state will be wrong. Using the ModifyAsync(Action<VxScene>, VxScene[]) method will schedule the modification not only to the right thread but also ensure the scene(s) to be in right state.
See also ./vx_how_to_develop.html#threading

Task ModifyAsync(VxSceneElement sceneElement, Action<VxScene> sceneModificationAction, params VxScene[] scenesToRead)

Parameters

sceneElement VxSceneElement

Any scene element of the scene to be modified.

sceneModificationAction Action<VxScene>

The (lambda) action containing the code to modify the scene

scenesToRead VxScene[]

An optional list of other scenes to be read. This scenes are also locked for reading in correct state.

Returns

Task

The awaitable Task for the modification

See Also
../guide/vx_how_to_develop.html#threading
