ModifyAsync(Action<VxScene>, params VxScene[])

It is highly recommended to use this method to modify the structure of a scene. A single modification will result in an internal re-calculation of many internal structures so it is a better approach to combine multiple modifications together. Additionally the modification of a scene requires the scene in "Idle" state. If you like to modify a the during the validation of a scene or node, the scene state will be wrong. Using the ModifyAsync(Action<VxScene>, params VxScene[]) method will schedule the modification not only to the right thread but also ensure the scene(s) to be in right state.
See also ./vx_how_to_develop.html#threading

public Task ModifyAsync(Action<VxScene> sceneModificationAction, params VxScene[] scenesToRead)
